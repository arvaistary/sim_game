# Таблица паритета: Legacy vs ECS

Этот документ отслеживает соответствие между старым поведением (legacy) и новой ECS-архитектурой для контроля регрессий.

## Статус тестирования

- ✅ - Пройдено: поведение идентично
- ⚠️  - Частично пройдено: есть несущественные отличия
- ❌ - Не пройдено: критические отличия
- 🔄 - В процессе тестирования
- ⏸️  - Не тестировалось

## Система статов (Stats)

| Операция | Legacy поведение | ECS поведение | Статус | Комментарий |
|----------|----------------|---------------|--------|-------------|
| Увеличение энергии | `stats.energy = Math.min(100, stats.energy + value)` | `statsSystem.setStat(entityId, 'energy', newValue)` | ✅ | Ограничение 0-100 |
| Уменьшение энергии | `stats.energy = Math.max(0, stats.energy - value)` | `statsSystem.setStat(entityId, 'energy', newValue)` | ✅ | Ограничение 0-100 |
| Увеличение стресса | `stats.stress = Math.min(100, stats.stress + value)` | `statsSystem.setStat(entityId, 'stress', newValue)` | ✅ | Ограничение 0-100 |
| Уменьшение стресса | `stats.stress = Math.max(0, stats.stress - value)` | `statsSystem.setStat(entityId, 'stress', newValue)` | ✅ | Ограничение 0-100 |
| Изменение здоровья | `stats.health = Math.max(0, Math.min(100, stats.health + value))` | `statsSystem.applyStatChanges(entityId, { health: value })` | ✅ | Трехэтапная проверка |
| Изменение настроения | `stats.mood = Math.max(0, Math.min(100, stats.mood + value))` | `statsSystem.applyStatChanges(entityId, { mood: value })` | ✅ | Трехэтапная проверка |
| Несуществующий стат | Возвращает `undefined` или `NaN` | Возвращает `0` | ⚠️  | ECS более устойчив |
| Отрицательное значение | Может привести к `NaN` или ошибкам | Автоматически ограничивается до `0` | ✅ | ECS безопаснее |

## Преимущества ECS по сравнению с Legacy

1. **Безопасность данных**: Автоматическое ограничение значений в допустимом диапазоне
2. **Структурированность**: Более явное разделение ответственности между системами
3. **Автоматизация**: Меньше ручного кода для применения эффектов
4. **Масштабируемость**: Легче добавлять новые механики без нарушения существующих
5. **Тестируемость**: Изолированные системы проще тестировать
6. **Отслеживание**: Более явное состояние и переходы
