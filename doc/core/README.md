# Game Life - Документация для разработчиков

Добро пожаловать в проект Game Life! Это симулятор жизни с пошаговым геймплеем, реализованный на Phaser.js.

## 📚 Быстрый старт

Для новых разработчиков рекомендуем следующий порядок чтения:

1. **🎯 Обзор проекта** - эта страница
2. **📊 Статус реализации** - что готово, что в работе
3. **🛣️ Roadmap разработки** - план развития
4. **🎮 GDD (Game Design Document)** - полное описание механик игры
5. **⚙️ Архитектура ECS** - техническая архитектура

## 📁 Структура документации

  - `09_hobbies.md` - хобби и побочный заработок
  - `10_achievements.md` - достижения и трофеи

## 🚀 Начало работы

### Требования

- Node.js (версия 16+)
- Современный браузер (Chrome, Firefox, Safari, Edge)

### Установка

```bash
# Клонирование репозитория
git clone <repository-url>
cd game_life

# Установка зависимостей (если есть package.json)
npm install
```

### Запуск проекта

```bash
# Запуск локального сервера
npm start
# или
python -m http.server 8000
```

Откройте `http://localhost:8000` в браузере.

## 🎯 Ключевые механики игры

### Core Loop

1. **Работа** - игрок нажимает "Начать рабочий период"
2. **Восстановление** - игрок тратит деньги на восстановление шкал
3. **Повтор** - цикл повторяется

### Шкалы персонажа

- Голод (Hunger)
- Энергия (Energy)
- Стресс (Stress)
- Настроение (Mood)
- Здоровье (Health)
- Физическая форма (Physical)

### Основные системы

- **Карьера** - progression через уровни работ
- **Образование** - книги, курсы, вузы
- **Дом** - жильё и мебель
- **Отношения** - семья и друзья
- **Инвестиции** - пассивный доход
- **Хобби** - побочный заработок

## 🔧 Технологический стек

- **Phaser.js 3** - игровой движок
- **ECS Pattern** - архитектура (Entity-Component-System)
- **JavaScript (ES6+)** - язык разработки
- **localStorage** - сохранение прогресса

## 🤝 Вклад в проект

### Для разработки новых функций

1. Изучите соответствующий модуль в GDD
2. Проверьте IMPLEMENTATION_STATUS.md для понимания контекста
3. Следуйте ECS архитектуре для новой логики
4. Добавьте тесты при необходимости

### Для исправления багов

1. Воспроизведите проблему
2. Найдите соответствующий код в `src/`
3. Проверьте ECS систем, которые могут влиять на проблему
4. Создайте фикс с тестом
