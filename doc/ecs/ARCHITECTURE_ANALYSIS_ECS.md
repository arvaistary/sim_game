### Структура ECS

#### Компоненты (Components)
Компоненты содержат данные без логики:

```javascript
// Примеры компонентов
{
  stats: {          // Ста игрока
    energy: 80,
    stress: 30,
    health: 90,
    mood: 70
  },
  career: {         // Карьера
    currentJob: { id: 'office_worker', salaryPerDay: 1500 },
    jobHistory: []
  },
  finance: {        // Финансы
    money: 10000,
    reserveFund: 2000,
    expenses: 3000,
    investments: []
  },
  // ... другие компоненты
}
```

#### Системы (Systems)
Системы содержат логику без данных:

```javascript
// Пример системы
class StatsSystem {
  applyStatChanges(entityId, changes) {
    const stats = world.getComponent(entityId, 'stats');
    
    for (const [key, value] of Object.entries(changes)) {
      stats[key] = Math.max(0, Math.min(100, stats[key] + value));
    }
    
    world.updateComponent(entityId, 'stats', stats);
  }
}
```

#### Адаптеры (Adapters)
Адаптеры обеспечивают интеграцию между ECS и существующим кодом:

```javascript
// SceneAdapter связывает сцену Phaser с ECS
class SceneAdapter {
  constructor(scene, saveData) {
    this.scene = scene;
    this.world = new ECSWorld();
    this.initializeFromSaveData(saveData);
  }
  
  getSystem(systemName) {
    return this.world.getSystem(systemName);
  }
  
  syncToSaveData() {
    // Синхронизация ECS компонентов в legacy формат
  }
}
```

## Решенные проблемы

### 1. Монолитный main.js ✅
**Было:** 3589 строк в одном файле с 10+ классами сцен
**Стало:**
- Отдельные файлы для каждой сцены в `src/scenes/`
- ECS системы в `src/ecs/systems/`
- Компоненты и адаптеры отдельно

### 2. Сильная связанность ✅
**Было:** Сцены напрямую импортируют функции из game-state.js
**Стало:**
- Сцены работают через `SceneAdapter`
- Логика инкапсулирована в ECS системах
- Минимальные зависимости через адаптеры

### 3. Дублирование кода ✅
**Было:** Похожие паттерны в разных сценах
**Стало:**
- Переиспользуемые ECS системы
- Единые паттерны через `SceneAdapter`
- Стандартизованные компоненты

### 4. Смешение ответственности ✅
**Было:** Сцены содержат и UI-логику, и бизнес-логику
**Стало:**
- Сцены отвечают только за UI
- Бизнес-логика в ECS системах
- Четкое разделение ответственности

### 5. Сложность навигации ✅
**Было:** Трудно найти нужный код среди тысяч строк
**Стало:**
- Логическая структура по системам
- Понятные имена файлов
- Карта домена для навигации

## Преимущества ECS архитектуры

### 1. Масштабируемость
```
Легко добавить новую фичу:

1. Создать компонент(ы) с данными
2. Создать систему с логикой
3. Подключить систему к ECSWorld
4. Использовать в сценах через SceneAdapter
```

### 2. Тестируемость
```
Системы изолированы и легко тестируются:

- 83 unit-теста для всех систем
- 10 smoke-тестов для сценариев
- 98.2% паритет с legacy
- Нет зависимости от UI или сцен
```

### 3. Переиспользуемость
```
Системы могут использоваться в разных контекстах:

- WorkPeriodSystem - в main.js и career scene
- RecoverySystem - в recovery scene и других сценах
- StatsSystem - везде где нужно изменить статы
```

### 4. Безопасность данных
```
Автоматическая валидация и ограничения:

- Ста всегда в диапазоне 0-100
- Деньги не могут быть отрицательными
- Структурированные компоненты вместо плоских объектов
```

### 5. Управление состоянием
```
Четкое разделение состояния:

- Компоненты = данные (чистые)
- Системы = логика (функции)
- Адаптеры = интеграция (границы)
```
