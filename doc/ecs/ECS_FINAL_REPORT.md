## 💡 Рекомендации по использованию ECS

### Для разработчиков:

**1. Инициализация сцен:**
```javascript
create() {
  this.saveData = loadSave();
  this.sceneAdapter = new SceneAdapter(this, this.saveData);
  this.sceneAdapter.initialize();

  // Получаем нужные системы
  this.timeSystem = this.sceneAdapter.getSystem('time');
  this.statsSystem = this.sceneAdapter.getSystem('stats');
}
```

**2. Использование систем:**
```javascript
// Работа
const summary = this.workPeriodSystem.applyWorkPeriodResult(5);

// Статы
this.statsSystem.applyStatChanges({ energy: -10, stress: 5 });

// Синхронизация
this.sceneAdapter.syncToSaveData();
persistSave(this, this.saveData);
```

**3. Получение данных:**
```javascript
// Получаем компоненты напрямую из world
const playerId = PLAYER_ENTITY;
const stats = this.world.getComponent(playerId, STATS_COMPONENT);
const career = this.world.getComponent(playerId, CAREER_COMPONENT);
```

### Для архитекторов:

**1. Принципы SOLID:**
- Каждая система должна иметь одну ответственность
- Системы должны зависеть только от компонентов
- Изменения в системах не должны ломать другие системы

**2. Паттерн Event-Driven Architecture:**
- Системы генерируют события при достижении условий
- Другие системы реагируют на события
- Расцепленная логика через EventQueueSystem

**3. Паттерн Repository:**
- GameStateAdapter выступает как репозиторий данных
- Обеспечивает кэширование и синхронизацию
- Изолирует логику сохранения от бизнес-логики
