## 3. Рекомендации по реализации в Phaser.js

### 3.1. Структура приложения

Используй состояния (state machine) или отдельные сцены для:

- **StartScene** — имя + возраст + выбор работы
- **MainScene** — отображение персонажа, шкал, кнопки «Работать»
- **RecoveryScene** — магазин + развлечения

### 3.2. Сохранение данных

Данные сохраняй в localStorage:

```javascript
{
  name: "Алексей",
  age: 25,
  gameDays: 87,
  money: 12450,
  hunger: 68,
  energy: 45,
  stress: 22,
  currentJob: { 
    name: "Офисный менеджер", 
    schedule: "5/2", 
    salaryPerWeek: 45000
  }
}
```

Для продвижения игрового времени веди счётчик gameDays и рассчитывай возраст как startAge + Math.floor(gameDays / daysPerYear).

### 3.3. Основные экраны игры

#### Главный экран (Main HUD)

- Персонаж (центр или слева, простой спрайт с эмоциями в зависимости от Настроения)
- Горизонтальные/вертикальные бары всех шкал (Голод, Энергия, Стресс, Настроение, Здоровье, Физическая форма)
- Индикаторы: Деньги, Возраст, Игровой день/месяц, Текущая работа + график
- Большая кнопка «Начать рабочий период» (главное действие)
- Кнопки быстрого доступа: Дом, Социальная жизнь, Образование, Финансы

#### Экран фазы восстановления (Recovery Screen)

Вкладки/разделы: Магазин (еда), Развлечения, Дом (мебель), Образование, Социальная жизнь, Финансы/Инвестиции

Все действия с предпросмотром влияния на шкалы (+/- иконки).

#### Экран работы / Интерактивное событие

Специальный полноэкранный режим с мини-игрой (кликер с таймером и огоньками) при срабатывании интерактивного события.

#### Экран образования (Школа / Институт)

Отдельные сцены с мини-играми для начального обучения.

#### Модальные окна

- Случайные события и события на работе (текст + 2–3 варианта выбора)
- Уведомления о повышении навыка, уровне дома, новом отношении
- Итоговый экран смерти / статистика жизни

### 3.4. Дополнительные HUD-элементы

- Маленький индикатор комфорта дома (иконка дома с цветом или процентом)
- Иконки активных эффектов (баффы от мебели, временные бонусы от событий)
- Кнопка «Пауза / Меню» с доступом к достижениям, навыкам, настройкам и сохранению

### 3.5. Общие требования к UI

- Минималистичный и чистый дизайн (flat или лёгкий pixel-art)
- Все важные действия должны быть в 1–2 клика
- Адаптивность под разные разрешения экрана (Phaser Scale Manager)
- Яркие цветовые индикаторы для шкал (зелёный → жёлтый → красный)
- Анимации tween для изменения баров и появления событий
