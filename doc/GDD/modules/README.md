## Краткое описание модулей

### 01_general.md
Общая информация об игре, жанр, платформа, ключевые особенности, а также основные механики игрового цикла (Core Loop).

### 02_implementation.md
Рекомендации по реализации в Phaser.js, структура приложения, сохранение данных, основные экраны игры, требования к UI.

### 03_core_mechanics.md
Все расширенные механики игры: случайные события, система навыков, события на работе, дом и обустройство, отношения, транспорт, здоровье, инвестиции, хобби, достижения, образование, начальное обучение, интерактивные события.

### 04_balance.md
Полный баланс и экономика игры: основные принципы, изменение шкал, таблицы работ и зарплат, расходы, система жилья и мебели, обучение, инвестиции.

### 05_save_system.md
Детальное описание системы сохранения: структура save-файла, точки автосохранения, слоты, экспорт/импорт, механика New Game+.

### 06_death_system.md
Система смерти и концовок: условия Game Over, финальный экран статистики, типы концовок, оценка жизни, действия после смерти.

### 07_random_events.md
Полный список из 25 случайных событий: положительные, отрицательные, нейтральные и возрастные события с описанием вероятностей и эффектов.

### 08_family.md
Расширенная механика семьи и детей: этапы отношений, романтические отношения, брак, дети, влияние на игру.

### 09_hobbies.md
Система хобби и побочного заработка: список хобби, уровни заработка, новые карьерные ветки.

### 10_achievements.md
Система достижений и трофеев: категории достижений (карьера, финансы, навыки, семья, возраст, жильё) и бонусы за достижения.

### 11_seasonal.md
Сезонные и праздничные события: праздники, возрастные переходы, временные события.

### 12_technical.md
Технические требования: архитектура приложения, производительность, звук и музыка, настройки игры.

### 13_roadmap.md
План разработки по версиям: 0.1 (MVP), 0.2, 0.3, 1.0, 1.1, 1.2+ с описанием задач и сроков.

### 14_conclusion.md
Заключение и итоговая информация о документе.

## Как использовать модули

### Для разработки:
1. Изучите `01_general.md` и `02_implementation.md` перед началом разработки
2. Ссылайтесь на соответствующие модули при обсуждении конкретных механик
3. Используйте `04_balance.md` для настройки всех числовых параметров игры
4. Следуйте `13_roadmap.md` для планирования разработки

### Для дизайна:
1. Изучите `02_implementation.md` для понимания UI/UX требований
2. Используйте `06_death_system.md` для создания финальных экранов
3. Ссылайтесь на `07_random_events.md` при создании визуальных эффектов событий

### Для тестирования:
1. Проверяйте все механики из `03_core_mechanics.md`
2. Тестируйте систему сохранения по `05_save_system.md`
3. Проверяйте баланс согласно `04_balance.md`

## Объединение модулей

Для объединения всех модулей в один файл GDD.md выполните:

```bash
cd game_life/doc/GDD/modules
cat 01_general.md 02_implementation.md 03_core_mechanics.md 04_balance.md 05_save_system.md 06_death_system.md 07_random_events.md 08_family.md 09_hobbies.md 10_achievements.md 11_seasonal.md 12_technical.md 13_roadmap.md 14_conclusion.md > ../GDD.md
```
