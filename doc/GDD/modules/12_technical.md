## 13. Технические требования

### 13.1. Архитектура приложения


Структура сцен Phaser.js:

```
Game (основной класс)
├── BootScene (загрузка ресурсов)
├── StartScene (ввод имени, выбор возраста, работа)
├── MainScene (основной игровой интерфейс)
├── RecoveryScene (магазин, развлечения, развитие)
├── EducationScene (школа, институт)
├── EventScene (случайные события)
├── WorkEventScene (события на работе, кликер)
├── DeathScene (финальная статистика)
└── SettingsScene (настройки, сохранения)
```

### 13.2. Производительность

Целевые показатели:
- FPS: 60 на десктопе, 30 на мобильных
- Время загрузки: < 3 секунды
- Размер бандла: < 2 MB (после оптимизации)

Оптимизации:
- Объектный пул для частиц и эффектов
- Ленивая загрузка ресурсов
- Сжатие изображений (WebP)
- Кэширование статических данных

### 13.3. Звук и музыка

Рекомендации по саундтреку:

**Фоновая музыка:**
- Главная тема: спокойная, вдохновляющая (120 BPM)
- Рабочая тема: ритмичная, мотивирующая (140 BPM)
- Фаза восстановления: расслабляющая (90 BPM)
- Финальная тема: трогательная, эпическая

**Звуковые эффекты:**
- Клик по кнопке: короткий и чёткий
- Получение денег: монетный звук
- Успешное действие: одобряющий звук
- Неудачное действие: мягкий отрицательный звук

### 13.4. Настройки игры


| Настройка | Опции |
|-----------|-------|
| Звук | Вкл/Выкл |
| Музыка | Вкл/Выкл, громкость 0–100 |
| Скорость игры | Нормальная, Быстрая, Очень быстрая |
| Сложность | Лёгкая, Нормальная, Сложная |
