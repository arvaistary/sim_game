# Game Design Document (GDD)

Это полный Game Design Document для проекта Game Life - симулятора жизни с пошаговым геймплеем.

## 📚 Обзор

GDD описывает все механики, системы и правила игры "Game Life". Документ организован модульно для удобства навигации и обновления.

## 📖 Рекомендуемый порядок чтения

Для нового разработчика рекомендуем следующий порядок:

1. **GDD.md** - основной документ (вся информация в одном месте)
2. **modules/** - детальные разделы по темам (при необходимости углубления)

Для быстрого старта достаточно прочитать только основной файл **GDD.md**.

## 🎯 Ключевые концепции

### Core Loop (Основной цикл)

1. **Работа** - игрок нажимает "Начать рабочий период"
2. **Восстановление** - игрок тратит деньги на восстановление шкал
3. **Повтор** - цикл повторяется

### Шкалы персонажа

- Голод (Hunger)
- Энергия (Energy)
- Стресс (Stress)
- Настроение (Mood)
- Здоровье (Health)
- Физическая форма (Physical)

### Основные системы

- **Карьера** - progression через уровни работ
- **Образование** - книги, курсы, вузы
- **Дом** - жильё и мебель
- **Отношения** - семья и друзья
- **Инвестиции** - пассивный доход
- **Хобби** - побочный заработок

## 🔧 Использование GDD

### Для разработчиков

1. Прочитайте основной файл **GDD.md** для понимания общей картины
2. Используйте модульные файлы для детального изучения конкретной темы
3. Ссылайтесь на GDD при реализации новых функций
4. Обновляйте GDD при изменениях механик

### Для дизайнеров

1. Используйте GDD как единый источник правды
2. Обновляйте соответствующие модули при изменениях
3. Следите за актуальностью баланса в разделе 04_balance.md
4. Добавляйте новые события в 07_random_events.md

## 🔄 Версионирование

GDD версия соответствует версии игры:

- **Версия 0.1 (MVP)** - базовый core loop
- **Версия 0.2** - дом + отношения + образование
- **Версия 0.3** - интерактивные события + инвестиции
- **Версия 1.0** - семья + хобби + достижения
- **Версия 1.1+** - оптимизация + баланс

## 📝 Внесение изменений

### Добавление новой механики

1. Найдите соответствующий модуль или создайте новый
2. Опишите механику в соответствующем разделе
3. Обновите основной файл **GDD.md**
4. Обновите статус реализации в `../core/IMPLEMENTATION_STATUS.md`
5. Уведомите команду об изменениях

### Изменение баланса

1. Обновите таблицы в `modules/04_balance.md`
2. Обновите основной файл **GDD.md**
3. Проверьте влияющие системы
4. Протестируйте изменения

## 🤝 Вклад

При внесении изменений в GDD:

1. Используйте понятный и простой язык
2. Добавляйте примеры где возможно
3. Обновляйте перекрёстные ссылки
4. Соблюдайте единый стиль форматирования
5. Увеличивайте версию при значительных изменениях
