# Обзор проекта Game Life

## Архитектурный обзор

Проект Game Life использует 4-х слойную архитектуру (Clean Architecture):

1. **Domain Layer** (`src/domain/`) - бизнес-логика и ECS
2. **Application Layer** (`src/application/`) - Use Cases  
3. **Infrastructure Layer** (`src/infrastructure/`) - внешние зависимости
4. **Presentation Layer** (`src/components/`, `src/pages/`, `src/composables/`) - Vue UI и Nuxt

## Ключевые системы

### Система обучения

Система обучения позволяет персонажу получать новые знания и прокачивать навыки через три типа образовательных активностей:

1. **Самостоятельное обучение** (Книги и онлайн-материалы)
   - Стоимость: 900+ руб
   - Длительность: 8ч
   - Эффект: +1 уровень навыка

2. **Онлайн-курсы и сертификации**
   - Стоимость: 6500+ руб  
   - Длительность: 24ч
   - Эффект: +2 уровня + небольшой буст к ЗП

3. **Учебные заведения** (вузы, колледжи)
   - Стоимость: 120000+ руб
   - Длительность: 160ч+
   - Эффект: +2+ уровня + диплом + сильный буст к карьере

Особенности обучения:
- Обучение происходит во время игрового времени и конкурирует с работой, отдыхом и сном
- Во время учебных действий применяются отдельные изменения шкал (усталость, стресс, настроение)
- Игрок может в любой момент прервать обучение (с потерей части прогресса)
- Прогресс обучения отображается в виде отдельного прогресс-бара
- С возрастом персонажа обучение становится сложнее
- Максимальный уровень навыка — 10
- Общее количество активных обучений одновременно — не более 1

### Система событий

Система случайных событий добавляет непредсказуемость и реиграбельность:

**Моменты срабатывания:**
- После завершения действия игрока (базовые шансы зависят от типа действия)
- В конце игровой недели (обязательно 1 событие)
- При достижении определённого возраста или количества игровых дней

**Типы событий:**
- Положительные (бонус к ЗП, + к шкалам, временный бафф навыка)
- Отрицательные (штраф к ЗП, − к шкалам, потеря здоровья)
- Нейтральные / сюжетные (дают выбор из 2–3 вариантов с разными последствиями)
- Возрастные (появляются после 30, 40, 50 лет — связаны со здоровьем или семьёй)

**Механика микро-событий:**
```
finalChance = baseChance(action,event) * riskModifiers * skillModifiers * stateModifiers
```
Где:
- `riskModifiers` — риск-факторы (дорогие вещи, опасный район, позднее время)
- `skillModifiers` — навыки и параметры (физическая форма, атлетизм)  
- `stateModifiers` — текущее состояние (усталость, стресс, здоровье)

### Система карьеры

Карьерная система определяет заработную плату и возможности роста:

**Стартовая экономика (возраст 18–22 года):**
- Курьер / Фрилансер: 12 000 – 18 000 руб/неделя
- Продавец / Официант: 22 000 – 28 000 руб/неделя  
- Офисный сотрудник: 35 000 – 45 000 руб/неделя
- Специалист (IT, инженер): 55 000 – 75 000 руб/неделя
- Менеджер среднего звена: 80 000 – 110 000 руб/неделя

**Рост ЗП:**
- Каждые 12–18 игровых месяцев возможен карьерный рост (+15–40% к ЗП)
- Каждый уровень навыка «Профессионализм» даёт +4% к ЗП
- Навык «Специализация» в соответствующей сфере даёт дополнительно +6–8%
- Максимальная ЗП на топовых должностях: 250 000 – 400 000 руб/неделя

**Карьерные действия:**
- Выполнить рабочий день (8ч) → Зарплата, Профессионализм +1
- Сверхурочная работа (3ч) → +30% ЗП за день, Стресс +10
- Поиск новой работы (4ч) → Шанс новой должности
- Подготовка к собеседованию (3ч) → Переговоры +2, Профессионализм +1
- Фриланс-заказ (6ч) → Деньги + Навык +2
- Смена карьеры (переобучение) (8ч) → Открытие новых веток
- Запросить повышение (1ч) → Шанс повышения +20%, Переговоры +1

### Другие ключевые системы

**Дом и обустройство:**
- Уровни жилья: Комната → Студия → 1-комнатная → 2-комнатная → Дом
- Покупка мебели даёт бонусы (хорошая кровать → −15% падения Энергии за рабочий период)

**Отношения и социальная жизнь:**
- Шкала отношений: от 0 до 100
- Действия: Встретиться с другом, Свидание, Позвонить родителям и т.д.
- Бонусы высокого уровня: Подарки, финансовая помощь, снижение стресса

**Здоровье и физическая форма:**
- Шкала «Физическая форма» влияет на общее Здоровье
- Действия: абонемент в спортзал, домашние тренировки, правильное питание

**Инвестиции и пассивный доход:**
- Варианты: банковский депозит, акции, собственный бизнес
- Автоматическое начисление пассивного дохода каждый игровой месяц

**Хобби и творчество:**
- Доступные хобби: кулинария, рисование, программирование, музыка, фотография, спорт
- Преимущества: восстановление Настроения, снижение Стресса, дополнительный заработок

## Техническая реализация

### ECS Архитектура

ECS (Entity-Component-System) используется для доменной логики:

**Компоненты (данные):**
- Время, статистика, навыки, кошелек, карьера, образование
- Жилье, мебель, отношения
- Финансы, инвестиции, кредиты, подписки  
- Очередь событий, история событий, журнал активности
- Статистика жизни, завершенные действия, перерывы

**Системы (логика):**
- TimeSystem - управление временем
- StatsSystem - статистика  
- SkillsSystem - навыки
- WorkPeriodSystem - рабочие периоды
- RecoverySystem - восстановление
- ActionSystem - система действий
- ActivityLogSystem - журнал активности
- CareerProgressSystem - прогресс карьеры
- FinanceActionSystem - финансовые действия
- InvestmentSystem - инвестиции
- MonthlySettlementSystem - ежемесячный расчёт
- EventQueueSystem - очередь событий
- EventChoiceSystem - выбор решений в событиях
- EventHistorySystem - история событий
- EducationSystem - образование
- PersistenceSystem - сохранение/загрузка
- MigrationSystem - миграция данных
- ChainResolverSystem - резолвер цепочек событий
- DelayedEffectSystem - отложенные эффекты
- LifeMemorySystem - система памяти жизни
- PersonalitySystem - система личности
- TagsSystem - система тегов
- SchoolSystem - система образования школы

### Интеграция слоёв

**Поток данных при действии пользователя:**
```
User Click → Vue Component → Composable → Pinia Store → Application Layer → Game Facade → ECS System → ECS World → Component Update → triggerRef → Vue Re-render
```

**Поток данных при запросе:**
```
Vue Component → Pinia Store → Computed Property → ECS World → getComponent → Component Data → Display in UI
```

## Руководство по разработке

### Добавление новой функции

1. **Domain Layer:**
   - Добавить компоненты ECS (если нужно)
   - Создать систему ECS для логики
   - Обновить Game Facade (команды/запросы)

2. **Application Layer:**
   - Добавить Use Case в commands.ts
   - Добавить запрос в queries.ts (если нужно)

3. **Presentation Layer:**
   - Создать Vue компонент страницы
   - Создать composable для логики UI
   - Добавить computed свойства в store

4. **Testing:**
   - Unit тесты для ECS систем
   - Unit тесты для Use Cases  
   - Integration тесты для компонентов

### Добавление новой страницы

1. Создать Vue компонент в `src/pages/`
2. Добавить маппинг в `src/nuxt-pages/game/[section].vue`
3. Создать composable для логики страницы (если нужно)
4. Добавить ссылку в навигацию MainPage
5. Обновить документацию `PAGES_REFERENCE.md`

### Добавление новой ECS системы

1. Создать файл системы в `src/domain/ecs/systems/`
2. Добавить систему в SystemContext
3. Реализовать логику в доменном слое
4. Добавить тесты в `test/unit/domain/ecs/`
5. Обновить документацию `ECS_DOMAIN_MAP.md`

## Текущее состояние

Проект мигрировал на Nuxt 4 + Vue 3 + TypeScript + Pinia с полноценной ECS архитектурой доменного слоя.
Основные механики игры реализованы: образование, события, карьера, дом, отношения, здоровье, инвестиции и хобби.
